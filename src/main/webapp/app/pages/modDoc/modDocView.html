Mod Panel

This is the mod panel you know?

Authorization

  • Before you can access the mode panel, you must be an Administrator on DevWars
  • If you are not already and Administrator, contact Terence Bezman and he’ll make you one.
  • We will soon implement an easier way to make new people administrators

Accessibility

  • You can click the moderate game button to head over to the moderation panel for specific game.
  • You may also click the settings button next to each game on the games page to access the mod panel for that specific game.

Create Game

  • Here you will be able to do all moderator related things.
  • Each tab is meant to be isolated from the rest so you can stick to one place.

Create Teams

  • Here you can click a player to add him or her to a team. A dialog will come up asking you which team and which language that they will be playing for.
  • You can remove a player from a team simply by clicking their name from either team. (They will not be penalized at this point)
  • Once finished, don’t forget to click the Save button at the bottom of the page.

Create Objectives

  • Here is where you can set the objectives before the game starts.
  • Simply enter the theme and objectives into the appropriate text boxes and click “Set Objectives”

Live Game

  • This is the page where you will spend most of your time.
  • Here you can penalise players for not showing up, change their statuses and mark which team won which objectives
  • As soon as SYNTAG starts the stream, click the activate game button. This will let the system know which data is relevant.
  • The team’s statuses are very important as they let the game host know what’s happening. If anything changes, update the status and click update.
  • Once the game is all over. The game host will begin to allow votes. Once voting ends, the person who is managing the Twitch Bot should let you know, it is at this point that you should click which team wins.
  • Note : If you accidentally click the wrong team, notify another Administrator.

Cloud 9

This is the editor where all player write their code. Go to c9.io and login as “devwars”. The password will be given to you by Steven or Beau. We have two workspaces, one for red and one for blue. We do not change these workspaces, because some tools (preview, codeview), depend on the url of the workspace. Before the game, open both workspaces. Make sure, that the HTML for the ZenGarden game is already uploaded, or you know when and how to upload it.

Adding New Users

To add a new user to the c9 workspace, click on Share in the top right. In the popup enter the c9-username and click on the Invite Button.

Make sure the “Notify people via email” is checked and you add the people with rw-access. Ask each player via teamspeak to write something in his file, so you can see, that they have correct access.

Remove Players from C9

When the game is over, disable the write access for each player. For that, click on the button in the Collaborate panel. Before adding the players for the next round, remove the old players. For that, click on the bin icon next to their name, to remove them.

Sass/SCSS

If players want to use SCSS, open a terminal (Alt-t) in the c9 workspace and execute: scss --watch style.scss:style.css If players will use css, disable the compiler by pressing CTRL-c in the terminal, where the command is running.

Saving Projects

After each game, back up the workspace. For this click on File -> Download Project and save the blue.tar.gz or red.tar.gz on your local machine. After the games the files should be uploaded to the Google Drive at Games > Game Archive > YYYY-MM-DD [Game NUMBER] If you don’t have access to the drive, send the files to Steven after the game.

Frame

Frame is the web app used to pull data from Firebase and dynamically update what viewers see on stream. Some things all Mods should know:

  • Frame can only read data from Firebase.
  • Frame has different URLs which are called Scenes. Steven is the only person that can switch Scenes on stream.
  • Each Scene may have multiple Stages.
  • Stage values are held in Firebase and are switched out by the DW_Bot commands.

Scenes & Stages

Scenes and Stages are very important. You should get familiar with them all (pictures will be added later):

  • Sidebar (/):
    • Objective Stage - Displays game timer, game theme, game objectives.
    • Voting Stage - This stage includes sub-stages (design, func, tiebreaker). Displays vote amounts for their respective category.
    • Betting Stage - Displays game timer, red and blue bet pools.
  • Pregame (/pregame): Displays current players and status for each team.
  • Blue (/blue): Displays HTML, CSS, and JS blue players. Used for classic games.
  • Red (/red): Displays HTML, CSS, and JS red players. Used for classic games.
  • Zen (/zen): Displays CSS players for both red and blue team.
  • Scoreboard (/scores): Displays points earned by each team.

Twitch Bot

DW_Bot is a nodejs server that reads Twitch chat. Some things all Mods should know:
  • DW_Bot can read and write to Firebase.
  • DW_Bot can read and write to DevWars Server.
  • DW_Bot can read and write to Twitch chat. This includes sending whispers.

Starting Game

To make sure the bot can connect to the server you can type !ping server. !startgame
  • Sets Frame to “objective” stage on firebase.
  • Resets voting on firebase.
  • Resets betting on firebase.
  • Reads current game info from server and updates firebase.
  • If the game type is “classic”, it will auto start bets in 5mins.
  • Start timer.
  • bot.say(‘Starting game!’).
  • NOTE: Steven Usually enters this command. Only enter this command once.

Betting

You should never need to put these commands in. Bets should start 5mins after !startgame and will close after 10mins. !openbets
  • bot.say(‘bets are open for game #’).
  • NOTE: Steven Usually enters this command. Only enter this command once.
!closebets
  • bot.say(‘bets will be closing soon’).
  • Reads current game info and saves json file with all bets.
  • bot.say(‘betting closed!’)
  • NOTE: Steven Usually enters this command. Only enter this command once.

Timer

You can’t pause the game but you can add/subtract time. !frame clock [minutes]
  • EXAMPLE: “!frame clock 10” will add 10 minutes. “!frame clock -5” will subtract 5 minutes.
!frame stage [stage]
  • EXAMPLE: !frame stage betting switches the Frame stage to betting.
  • NOTE: Stage options: objective, betting, voting.

Applying

!applyfor [gameID]
  • NOTE: The command that users will use is !apply.

Voting

There are 3 different voting categories; design, functionality, tiebreaker. !startvote [category]
  • EXAMPLE: !startvote f (shorthand) or !startvote functionality will open votes for functionality.
  • NOTE: shorthand options are: d, f, t.

Ending Game

!winner [teamcolor]
  • Get current game ID from server and then use that to load the json file that holds bets for each team. This is used to calculate winnings.
  • bot.say(‘blue pool:’) bot.say(‘red pool:’).
  • Sends out whisper (every second) to each user who betted.
  • EXAMPLE: If blue team wins type “!winner blue”.
  • NOTE: This command is for classic games only and needs to be done before deactivating the game on the mod panel.
!endgame
  • Resets voting on firebase.
  • Resets betting on firebase.
  • Update timer to ‘false’.
  • Sets Frame to “objective” stage on firebase.
